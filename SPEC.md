# thermal report generator — 開發規格書

> 建立日期：2026/2/27

---

## Phase 0：需求定義

- **一句話描述**：這工具可以讓工程師輸入實驗得到的數據及資料(照片)等等訊息，透過工具快速輸入的介面把資料導入，工具自動整理後，生成專業的thermal report
- **使用者**：thermal RD
- **觸發情境**：每次做完實驗要整理報告時
- **輸入**：1.溫度記錄器的紀錄資料，格式會是CSV檔案
2.PCB板、產品設備表面的IR照片
3.架設實驗完的照片
4.各個有量測的PCB板照片，上面會標注有黏thermal couple 的monitor point
- **輸出**：1.封面：案子名稱、日期、實驗者、案子的stage、或是你建議給我改登入的資訊
2.按照我輸入的資料，妳以專業的寫報告高手來分配每頁該呈現的內容
- **成功定義**：原本整理一個報告需要一個多小時，縮短到10分鐘

---

## Phase 1：規格設計

### 1.1 資料格式規格
- **輸入檔案格式**：.csv、手動輸入、其他
- **輸入欄位定義**：
  - 報告內容主要分為幾頁
  - 1.案子的基本資料訊息、實驗設備、軟體版本、測試設備照片
  - 2.該案所被測試的照片，照片放入後可以點擊照片中元件的位置來新增該元件的名稱（用拉引線的方式）
  - 3.讀入溫度記錄器的csv檔案，自動產生表格，溫度部分主要分為常溫和高溫兩個測試條件，每個元件會有名稱、溫度規格、derating溫規、該元件使用的TIM、TIM的K值、測試得到的溫度與derating溫規的差值（即溫度margin)
- **輸出格式**：多種格式

### 1.2 計算邏輯規格
無

### 1.3 邊界條件處理
無

### 1.4 技術架構
- **前端框架**：Reflex
- **資料庫**：Firebase
- **介面需求**：需要 UI
- **程式語言**：Python
- **版本控制**：GitHub
- **部署**：Reflex Cloud

### 1.5 限制與假設
所有欄位盡可能用下拉式選單，盡量減少文字輸入方式

## Phase 2：功能拆解

1. 該工具要有可以儲存的資料庫支援本地輸出儲存與雲端儲存
2. 溫度紀錄的表單第三頁能夠有可以新建元件資料的欄位
3. AI分析報告的總結功能

---

## Phase 3：開發規則

**一次只做一個功能，測試通過再做下一個。**

每輪開發使用以下格式：
- 【目標】這一步要完成什麼
- 【輸入】這個功能接收什麼
- 【輸出】這個功能要產出什麼
- 【規則】計算邏輯或判斷條件
- 【範例】一組具體的輸入 → 預期輸出

---

## Phase 4：測試驗證

| 測試類型 | 內容 |
|---|---|
| 正常測試 | 用標準輸入驗證結果正確性 |
| 邊界測試 | 用極端值測試（最大、最小、零、空白） |
| 錯誤測試 | 輸入錯誤資料（格式不對、缺欄位） |
